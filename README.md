# pastry-shop

Проект на Django для керування контентом сайту.

## Встановлення та запуск

1. Встановіть залежністі:
```bash
pip install -r requirements.txt
```

2. Застосуйте міграції:
```bash
python manage.py migrate
```

3. Запустіть сервер розробки:
```bash
python manage.py runserver
```

4. Відкрийте сайт у браузері: http://127.0.0.1:8000/

## Структура проекту

- `shop/` - основний додаток, що відповідає за роботу магазину (асортимент, замовлення, сторінки сайту).
    - `admin.py` - налаштування адміністративної панелі для додатку `shop`.
    - `apps.py` - конфігурація додатку.
    - `forms.py` - форми Django для додатку `shop` (наприклад, форма замовлення).
    - `migrations/` - папка з файлами міграцій бази даних для додатку `shop`.
    - `models.py` - моделі даних додатку `shop` (Асортимент, Замовлення).
    - `templates/shop/` - шаблони HTML для додатку `shop`.
    - `urls.py` - URL-адреси додатку `shop`.
    - `views.py` - представлення (views) для додатку `shop`, що обробляють запити користувачів.
- `my_bakery/` -  папка з конфігурацією проєкту Django.
    - `settings.py` - основні налаштування Django проєкту (база даних, шляхи, тощо).
    - `urls.py` - головний файл URL-адрес проєкту, що включає URL-адреси додатків.
- `static/` - папка для статичних файлів:
    - `css/` - файли CSS стилів.
    - `img/` - зображення.
- `media/` - папка для медіафайлів, завантажених користувачами або адміністратором сайту.
    - `assortment/` - зображення для товарів асортименту.
- `templates/` - загальна папка для шаблонів (наразі не використовується, шаблони додатку `shop` знаходяться в `shop/templates/shop/`).

## Технології

- **Django** - Python веб-фреймворк
- **Bootstrap 5** - CSS-фреймворк для адаптивного дизайну
- **SQLite** - база даних (використовується за замовчуванням для розробки)
- **python-dotenv** - для управління змінними оточення (безпечне зберігання секретів)
- **django-crispy-forms** - для покращення відображення форм Django з Bootstrap 5

## Встановлення та налаштування

1. **Налаштування середовища**
   - Запустіть файл `setup_env.bat`
   - Це створить віртуальне середовище та встановить всі необхідні залежності
   - Дочекайтесь завершення встановлення

2. **Налаштування бази даних**
   - Запустіть файл `reset_db.bat`
   - Це створить нову базу даних та застосує всі міграції

3. **Створення адміністратора**
   - Запустіть файл `create_superuser.bat`
   - Введіть дані адміністратора:
     * Ім'я користувача (username)
     * Email адреса
     * Пароль (двічі)
   - Запам'ятайте ці дані, вони потрібні для входу в адмін-панель

4. **Робота з сервером**
   - Для запуску: запустіть файл `run_server.bat`
   - Сайт буде доступний за адресою: http://127.0.0.1:8000/
   - Адмін-панель доступна за адресою: http://127.0.0.1:8000/admin/
   - Для зупинки: запустіть файл `stop_server.bat`

## Використання адмін-панелі

1. **Вхід в адмін-панель**
   - Перейдіть за адресою http://127.0.0.1:8000/admin/
   - Введіть логін та пароль адміністратора

2. **Можливості адмін-панелі**
   - Управління кондитерами
   - Перегляд та обробка замовлень
   - Створення нових користувачів
   - Налаштування прав доступу

## Основні функції сайту

1.  **Головна сторінка ("Головна")**
    - Вітальна інформація про кондитерську "Sugar Bliss".
    - Короткий огляд бестселерів асортименту.
    - Посилання на сторінку "Асортимент", "Про нас" та форму замовлення.

2.  **Перегляд асортименту ("Асортимент")**
    - Повний перелік кондитерських виробів з назвами, зображеннями та цінами.
    - Можливість перейти до форми замовлення для кожного товару.

3.  **Сторінка "Про нас" ("Про нас")**
    - Інформація про історію кондитерської, філософію та команду.
    - Контактна інформація (адреса, телефон, email, графік роботи).

4.  **Форма замовлення ("Зробити замовлення")**
    - Форма для оформлення замовлення на кондитерські вироби.
    - Поля для вибору товару з асортименту, кількості, дати замовлення та доставки, контактних даних замовника та коментаря.
    - Валідація введених даних.
    - Відправка повідомлення про замовлення на електронну пошту адміністратора.
    - Сторінка підтвердження успішного замовлення.

5.  **Адміністративна панель (Django Admin)**
    - Керування асортиментом (додавання, редагування, видалення товарів).
    - Перегляд та редагування замовлень.
    - Можливість змінювати статус замовлення.

## Інструменти розробника (Django Debug Toolbar)

### Що це таке?
Django Debug Toolbar (DjDT) - це панель інструментів для відладки, яка з'являється справа на сторінці під час розробки. Вона надає детальну інформацію про роботу вашого Django додатку.

### Як увімкнути/вимкнути?
1. **Увімкнення:**
   - В файлі `.env` встановіть `DJANGO_DEBUG=True`
   - Перезапустіть сервер

2. **Вимкнення:**
   - В файлі `.env` встановіть `DJANGO_DEBUG=False`
   - Перезапустіть сервер

1. **Versions (Версії)**
   - Версії Python та Django

2. **Timer (Таймер)**
   - Час завантаження сторінки
   - Час виконання компонентів

3. **Settings (Налаштування)**
   - Поточні налаштування Django

6. **SQL**

   - Аналіз продуктивності

7. **Static files (Статичні файли)**
   - Використані статичні файли
   - Шляхи до файлів

8. **Templates (Шаблони)**
   - Використані шаблони
   - Контекст шаблонів
   - Час рендерингу

## Управління контентом сайту

### Доступ до панелі управління контентом

1. **Вхід в адмін-панель**
   - Відкрийте http://127.0.0.1:8000/admin/
   - Введіть логін та пароль адміністратора
   - У меню зліва знайдіть розділ "Управління контентом"

### Налаштування сайту (Site Settings)

1. **Загальні налаштування**
   - Перейдіть до розділу "Налаштування сайту"
   - Натисніть "Додати налаштування сайту" (можна створити тільки один запис)
   - Заповніть поля:
     * "Назва сайту" - назва, що відображатиметься у заголовку
     * "Логотип" - завантажте логотип (опціонально)
     * "Favicon" - завантажте іконку сайту (опціонально)
     * "Текст футера" - текст, що відображатиметься внизу сторінки

2. **Робота з зображеннями**
   - Підтримувані формати: JPG, PNG, GIF
   - Рекомендовані розміри:
     * Логотип: 200x50 пікселів
     * Favicon: 32x32 пікселів
   - Зображення автоматично зберігаються у папці `media/site_images/`

### Управління сторінками (Pages)

1. **Створення нової сторінки**
   - Перейдіть до розділу "Сторінки"
   - Натисніть "Додати сторінку"
   - Заповніть поля:
     * "Заголовок" - назва сторінки
     * "URL" - генерується автоматично з заголовку
     * "Контент" - вміст сторінки (текст, HTML)
     * "Зображення" - ілюстрація до сторінки (опціонально)
     * "Опубліковано" - статус публікації

2. **Редагування сторінок**
   - У списку сторінок натисніть на потрібну
   - Внесіть необхідні зміни
   - Натисніть "Зберегти"

3. **Управління публікацією**
   - Можна тимчасово приховати сторінку
   - Зніміть галочку "Опубліковано"
   - Сторінка буде недоступна для відвідувачів

### Оголошення (Announcements)

1. **Створення оголошення**
   - Перейдіть до розділу "Оголошення"
   - Натисніть "Додати оголошення"
   - Заповніть поля:
     * "Заголовок" - тема оголошення
     * "Зміст" - текст оголошення
     * "Зображення" - картинка (опціонально)
     * "Активно" - статус показу

2. **Управління оголошеннями**
   - Можна редагувати існуючі оголошення
   - Можна тимчасово приховати (зняти галочку "Активно")
   - Оголошення сортуються за датою створення

### Поради з використання

1. **Робота з текстом**
   - Використовуйте форматування тексту
   - Можна додавати HTML-теги
   - Підтримується копіювання тексту з Word

2. **Робота з зображеннями**
   - Використовуйте оптимізовані зображення
   - Рекомендований формат: JPG для фото, PNG для графіки
   - Максимальний розмір файлу: 5 МБ

3. **Безпека**
   - Регулярно зберігайте зміни
   - Не передавайте дані для входу стороннім
   - Створюйте резервні копії контенту

### Технічні деталі

1. **Обмеження**
   - Максимальна довжина заголовку: 200 символів
   - Підтримувані формати зображень: JPG, PNG, GIF
   - Автоматичне видалення старих файлів при заміні

2. **Автоматизація**
   - URL генерується автоматично з заголовку
   - Зображення автоматично оптимізуються
   - Старі файли автоматично видаляються

3. **Відстеження змін**
   - Система зберігає дату створення
   - Відслідковується дата останнього оновлення
   - Історія змін доступна в адмін-панелі
